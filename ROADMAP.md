## 一、项目概览

### 一句话描述
让考公用户在对话中完成选岗、申论批改、面试模拟，而不是在传统培训 app 里点点点。

### 双轨策略

| 轨道 | 形态 | 目标 | 变现 |
|---|---|---|---|
| **开源线** | `gwy-mcp` Python MCP Server（AGPL-3.0） | 建社区声量、占领 GitHub 关键词、获取开发者用户 | 免费 |
| **商业线** | 桌面应用（Tauri + React） | 面向真实考公用户的一站式工具 | License Key 售卖 |

### 目标用户
- 大专/本科/研究生学历，22-35 岁
- 能独立安装桌面软件，不熟悉命令行/编程
- 每年国考+省考+事业编报名季集中使用（10月-次年3月为高峰）
- 付费意愿：线下培训班 3000-20000 元/期，线上批改 50-300 元/篇

### 为什么桌面应用而非 Web SaaS
1. 本地语音处理延迟低（面试模拟场景对延迟敏感）
2. 离线能力（选岗数据可缓存，部分功能无需网络）
3. License Key 本地校验更安全
4. 中国用户对"安装软件"的接受度远高于"配置开发者工具"
5. Tauri 打包体积小（~10MB vs Electron ~120MB），更新推送不依赖应用商店

## 四、Phase 4：桌面应用 MVP

**预计时间：** 4-6 周
**目标：** 让用户能双击打开应用，使用选岗匹配和申论批改功能

### 4.1 Tauri + React 项目初始化

- 初始化 Tauri 2.x 项目，配置 React 18 + TypeScript
- 设定窗口尺寸、菜单栏、系统托盘
- 集成 MCP Server 为进程内调用（非 stdio 通信，直接 import Python 模块）

### 4.2 选岗匹配界面

- **公告导入页**：拖拽 PDF / 粘贴 URL / 粘贴文本 三种方式
- **岗位列表页**：表格展示提取的岗位（单位名称、专业要求、学历要求、招录人数）
- **匹配结果页**：可报/不可报标签、不满足条件逐条展示、按匹配度排序
- **个人画像设置页**：表单填写（学历/专业/应届/基层/政治面貌/户籍）

### 4.3 申论批改界面

- **作答提交页**：选择题型、粘贴题目+作答、提交批改
- **批改结果页**：总分+等级卡片、六维评分柱状图、逐段批注（原文+建议对照）、改进优先级列表
- **范文参考**：查看同题型高分范文

### 4.4 离线能力

- 选岗数据缓存到本地 SQLite
- 用户画像本地持久化
- 批改历史记录

## 六、Phase 6：AI 面试模拟

**预计时间：** 6-8 周（申论批改稳定后启动）
**目标：** 高客单价差异化功能

### 6.1 结构化面试模拟（单人）

- 用户选择岗位类型（综合/执法/乡镇/技术）
- AI 出题（文字 + 语音播报）
- 限时思考（40秒-2分钟）+ 限时作答（3-5分钟）
- 语音转文字 → 五维评分：
  - 内容完整度（是否踩到答题要点）
  - 逻辑结构（总分总/是什么为什么怎么办/主体分析法）
  - 政策高度（是否引用政策文件）
  - 语言表达（流畅度、口头禅检测）
  - 仪态建议（语速、停顿、音量变化）
- 逐句分析 + 参考回答展示

### 6.2 无领导小组讨论模拟（多人 Multi-Agent）

- 6-10 个 AI Agent 模拟考生，用户参与其中
- 每个 Agent 有独立性格模板（激进型/沉稳型/逻辑型/跑题型）
- 讨论流程：个人陈述 → 自由讨论 → 总结陈词
- AI 考官全程记录：发言占比统计、有效贡献分析、逻辑链复盘

**此功能为壁垒级——目前市场上没有同类产品。**

## 九、竞争分析

### GitHub 开源竞品

| Repo | Stars | 实质内容 |
|---|---|---|
| `xuebazo/gongkao_ai` | ⭐1 | README 驱动开发 |
| `AIDEWWJ/shenlun-agent` | ⭐2 | 裸 prompt + API 调用 |
| `LearnTechLogic/shenlun-ai` | ⭐1 | 同上 |
| `luchen123456/shenlun-ai` | ⭐0 | 同上 |
| `mkih76/SLB` | ⭐0 | 同上 |

**关键发现：** GitHub 上无人在做结构化评分 + 分题型 rubric + 范文锚定 + 选岗批改联动。所有竞品都是"一个 prompt + API 调用 = 一段文本输出"。

### 商业竞品

| 竞品 | 优势 | 劣势 |
|---|---|---|
| 粉笔/中公/华图 | 品牌+用户+题库 | AI 做得浅，不敢革自己人工批改的命 |
| AI 写作工具（秘塔/文心） | 通用 AI 能力强 | 没有公考领域知识，不会分题型评分 |
| DeepSeek 官方 prompt | 免费、方便 | 无结构化输出、无范文锚定、无逐段批注 |

### gwy-mcp 的差异化壁垒

1. **选岗 + 批改 + 面试 = 全流程覆盖**（无人做到）
2. **分题型评分细则 + 范文锚定**（非裸 API 调用）
3. **MCP Server 形态**（可被 Claude Desktop / Codex 直接调用）
4. **开源 + 商业双轨**（社区声量 + 商业变现）
