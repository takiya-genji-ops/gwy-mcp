# gwy-mcp 项目规划文档

> 最后更新：2026-06-08
> 定位：AI 驱动的公务员考试智能助手，开源 MCP Server + 商业桌面应用双轨并行

---

## 一、产品定位

### 一句话描述
让考公用户在对话中完成选岗、申论批改、面试模拟，而不是在传统培训 app 里点点点。

### 双轨策略
| 轨道 | 形态 | 目标 | 变现 |
|---|---|---|---|
| **开源线** | `gwy-mcp` Python MCP Server | 建社区声量、获 GitHub star、占领关键词 | 免费 |
| **商业线** | 桌面应用（Tauri + React） | 面向真实考公用户的一站式工具 | License Key 售卖 |

### 目标用户画像
- 大专/本科/研究生学历，22-35 岁
- 能独立安装桌面软件，不熟悉命令行/编程
- 每年国考+省考+事业编报名季集中使用
- 付费意愿：线下培训班 3000-20000 元/期，线上批改 500-3000 元/期

## 二、核心功能路线图

### V1：选岗 + 结构化面试（MVP，预计 6-8 周）

#### 2.1 公告解析与选岗匹配（免费，获客入口）
```
用户输入：
  - 省份/考试类型（江苏省考/江苏事业编/国考...）
  - 个人条件（专业、学历、应届/往届、基层经历、户籍等）

系统行为：
  1. 接收官方公告（URL 或 PDF）
  2. LLM 结构化提取岗位信息（单位名称、招录人数、专业要求、学历要求...）
  3. 逐岗位匹配用户条件
  4. 输出可报岗位列表 + 竞争比估算 + 历年进面分数参考

输出示例：
  "共筛选出 12 个可报岗位，竞争比最低的是 XX 局科员（预计 15:1），
   去年同岗位进面分 132.5，建议优先考虑。"
```

**数据来源（分阶段）：**
- 阶段 A（V1 先期）：用户粘贴公告链接/上传 PDF，LLM 解析
- 阶段 B（V1 后期）：后台定时任务抓取重点省份人社局/组织部公告
- 明确不爬：公考雷达、粉笔等商业平台

#### 2.2 结构化面试模拟（付费，第一收入点）
```
流程：
  1. 用户选择岗位类型（综合/执法/乡镇/技术）
  2. AI 从对应题库中出题（文字展示 + 语音播报备选）
  3. 用户限时思考（40 秒-2 分钟，根据题型）
  4. 用户限时作答（3-5 分钟），语音输入转文字
  5. AI 从 5 个维度打分：内容完整度、逻辑结构、政策高度、语言表达、仪态建议
  6. 逐句批改 + 参考范文展示
```

**定价：** 每次完整模拟 9.9 元；月卡 49 元无限次

### V2：申论批改 + 无领导面试（预计 V1 后 8-12 周）

#### 2.3 申论智能批改（付费，核心收入点）
```
流程：
  1. 用户提交申论作答（文字输入）
  2. AI 从 5 个维度评分：立意准确度、论证深度、对策可行性、格式规范、语言表达
  3. 分段落批注修改建议
  4. 提供 3 篇不同分数档的参考范文
  5. 生成"薄弱维度雷达图"和针对性训练建议
```

**定价：** 按篇收费 5-9 元；月卡 39-69 元

#### 2.4 无领导小组讨论模拟（差异化壁垒，高价）
```
场景：6-10 名 AI Agent 模拟考生，用户参与其中
  - 每个 AI Agent 有独立性格模板（激进型/沉稳型/逻辑型/跑题型/沉默型）
  - 讨论流程：个人陈述 -> 自由讨论 -> 总结陈词
  - 讨论过程中 AI 考官观察记录
  - 结束后：发言占比统计、有效贡献分析、逻辑链条复盘、改进建议

技术要点：
  - Multi-Agent 协调（每个 Agent 是独立 LLM 调用）
  - 实时对话节奏控制（抢话、让步、引导等行为模拟）
  - 全程录音转文字 + 时间轴标记
```

**定价：** 每次 29-69 元（竞品真空，可定高价）

### V3：备考管理（预计 V2 后 6-8 周）
- 个人薄弱点诊断（基于历史作答数据）
- 每日学习计划生成
- 时政热点智能摘要（自动关联申论考点）
- 考前预测（基于历年真题规律 + 当年政策热点）

### 整体架构图
```

|  +----------+ +----------+ +-----------+  |

```

### 技术选型

| 层 | 技术 | 理由 |
|---|---|---|
| **MCP Server** | Python 3.11+ / FastMCP | Python 在 LLM 生态最成熟；FastMCP 开发效率高 |
| **桌面应用** | Tauri 2.x / React 18 / TypeScript | 打包体积小（~10MB vs Electron ~120MB）；Windows 优先 |
| **云后端** | FastAPI + PostgreSQL + Redis | 轻量、异步、Python 生态统一 |
| **LLM** | Claude API（主力）/ DeepSeek API（备选） | Claude 中文能力 + 指令遵循好；DeepSeek 成本更低 |
| **语音** | Web Speech API（Chrome/Edge 内置） | 零成本，考公用户以 Windows + Edge 为主 |
| **支付** | 支付宝当面付 / 微信 JSAPI | 覆盖中国用户支付方式 |

### 为什么桌面应用而非纯 Web SaaS
1. 本地语音处理延迟低（面试模拟场景对延迟敏感）
2. 离线能力（选岗数据可缓存，部分功能无需网络）
3. License Key 本地校验更安全
4. 中国用户对"安装软件"的接受度远高于"配置开发者工具"
5. Tauri 打包体积小，更新推送不依赖应用商店

## 四、数据模型

### 核心实体
```
User（用户）
  - id, phone, wechat_union_id
  - 个人画像：{学历, 专业, 毕业年份, 户籍, 基层经历, 政治面貌}
  - license_key, license_expiry
  - 使用次数统计

Exam（考试公告）
  - id, title, province, exam_type（国考/省考/事业编/选调生）
  - announce_url, announce_date, exam_date
  - status（报名中/已截止/即将开始）

Position（岗位）
  - id, exam_id, unit_name（用人单位）, position_name
  - recruitment_count（招录人数）
  - requirements: {学历, 专业列表, 应届要求, 基层经历, 政治面貌, 户籍, 其他}
  - salary_level, work_location
  - history_entries: [{year, applicants, entry_score}]

InterviewSession（面试记录）
  - id, user_id, position_type
  - question, user_answer, ai_score, ai_feedback
  - duration, created_at

EssaySubmission（申论提交）
  - id, user_id, topic, user_content
  - ai_score, dimension_scores, paragraph_feedback
  - created_at

LicenseKey（激活码）
  - key, type（月卡/年卡/永久）, status, activated_at, expires_at
  - user_id（激活后关联）
```

## 六、风险分析

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| 答案准确率不足（行测题） | 高 | 口碑崩 | V1 只做选岗+面试，避开行测客观题 |
| 题库版权纠纷 | 中 | 法律风险 | V1 不给原题，只做批改和模拟题生成 |
| 政府网站反爬/改版 | 高 | 数据断供 | V1 靠用户提供公告，后续小范围自动采集 |
| 竞争（粉笔/中公上线 AI 功能） | 中 | 流量争夺 | 差异化在 Multi-Agent 面试，非大厂核心方向 |
| LLM 幻觉导致不准确岗推 | 中 | 用户投诉 | 每个推荐结果附带原始公告引用链接 |
| 用户不会用/不想装桌面软件 | 中 | 付费转化低 | 同时提供微信小程序版本作为降级方案 |

## 八、开放问题（待决策）

1. **DeepSeek vs Claude 主力模型？** DeepSeek 成本低、国内访问稳定，但中文指令遵循和结构化输出能力 Claude 更强。建议 V1 用 Claude，量大了切 DeepSeek。
2. **语音方案？** Web Speech API 免费但 Edge 限定；讯飞 API 效果好但增加成本。建议 V1 用 Web Speech API 跑通，后续可选接讯飞。
3. **开源的 MCP Server 开哪些 Tool？** 建议开源选岗匹配、面试模拟的评分逻辑，但不开放 License 校验、支付等商业模块。
4. **要不要先做一个微信小程序 POC 测试需求？** 开发成本低，可快速验证用户是否愿意为 AI 面试付费。
